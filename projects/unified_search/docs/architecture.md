# 统一搜索服务架构设计

## 概述

统一搜索服务是一个集成多个搜索提供商的中间层服务，旨在提供高可用性和智能路由能力。该服务主要面向OpenClaw智能体系统，为其提供可靠的网络搜索功能。

## 架构组件

### 1. 搜索提供商适配器
- **Exa API适配器**：负责与Exa搜索API交互
- **Tavily API适配器**：负责与Tavily搜索API交互
- 提供统一的接口抽象，使上层逻辑无需关心底层提供商的具体实现

### 2. 智能路由引擎
- 根据预设的场景优先级表选择搜索提供商
- 支持多种搜索场景：
  - CHINA_SOCIAL：中国社交相关内容搜索
  - GLOBAL_TRENDS：全球趋势搜索
  - TECH_RESEARCH：技术研究搜索
  - GENERAL：通用搜索
- 提供故障转移机制，当前提供商不可用时自动切换到备用提供商

### 3. 结果聚合器
- 标准化不同提供商返回的数据格式
- 统一结果表示，确保前端或其他服务可以一致地处理搜索结果

### 4. HTTP服务层
- 提供RESTful API接口
- 接收JSON格式的搜索请求
- 返回Markdown格式的搜索结果

## 数据流

1. 客户端发送POST请求到`/`端点，包含查询参数和场景类型
2. HTTP服务层接收请求并解析参数
3. 智能路由引擎根据场景确定提供商优先级
4. 依次尝试每个提供商直到获得有效结果或所有提供商都失败
5. 结果聚合器标准化搜索结果
6. HTTP服务层将结果转换为Markdown格式并返回给客户端

## 错误处理

- 当提供商API返回错误时，服务会尝试下一个优先级的提供商
- 如果所有提供商都失败，返回包含错误信息的降级结果
- 详细的错误日志记录，便于运维和调试

## 配置

- 服务端口可通过环境变量`SEARCH_SERVICE_PORT`配置（默认18790）
- 密钥文件路径可通过环境变量`SECRETS_PATH`配置（默认为OpenClaw标准路径）
- 支持运行时动态加载密钥，无需重启服务

## 安全考虑

- API密钥存储在独立的密钥文件中，不在代码中硬编码
- 服务仅监听本地回环地址（127.0.0.1），防止外部直接访问
- 输入参数验证，防范注入攻击
